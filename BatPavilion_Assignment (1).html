What is Full Stack Development?

Full stack development is the practice of building both sides of a web application — the front-end (everything the user sees in their browser) and the back-end (the server, database, and application logic running behind the scenes). A developer who works across both layers is called a Full Stack Developer.

The front-end is built with HTML (page structure), CSS (visual styling), and JavaScript (interactive behaviour). The back-end typically uses a server language such as Node.js or Python, connected to a database like MongoDB or PostgreSQL, exposed via an API that the front-end calls to fetch or save data.

For Bat Pavilion, the front-end is this complete HTML/CSS/JS/Bootstrap website. In a real production deployment, the back-end would handle user accounts, bat inventory stored in a database, custom order processing, and secure payment integration — all invisible to the shopper but essential to the business.

🖥 Front-End:  HTML · CSS · JavaScript · Bootstrap Browser
⚙️ Back-End:  Node.js · Express · REST API Server
🗄 Database:  MongoDB (Products, Orders, Users) Data Layer

HTML — The Structure of the Web

HTML (HyperText Markup Language) is the foundational language of every web page. It defines the structure and content of a page using elements (tags), which browsers interpret and render into what you see on screen. Every heading, paragraph, image, link, and button on a page is an HTML element.

HTML5 introduced semantic elements — tags whose names communicate meaning beyond just layout. For example, <nav> tells the browser "this is navigation", <section> marks a distinct content area, and <footer> marks the page footer. This improves accessibility for screen readers and helps search engines understand the page structure (SEO).

In Bat Pavilion, semantic HTML is used throughout — <nav> for the navigation bar, <section> for each content block (hero, bats, order, reviews, ePortfolio), <form> for the custom order form with required validation, and proper alt attributes on every product image for accessibility.

Semantic Tags Forms & Inputs alt attributes Anchor links Lists (nav) Meta viewport Headings h1–h6 HTML5 Validation

CSS — Styling the Web

CSS (Cascading Style Sheets) is the language used to control the visual presentation of HTML elements — their colours, fonts, spacing, layout, and animations. Without CSS, every web page would look like an unstyled document.

CSS works through a cascade — styles are applied based on specificity and source order, allowing you to override defaults selectively. Modern CSS includes powerful layout tools: Flexbox for one-dimensional layouts (rows or columns) and CSS Grid for two-dimensional layouts (rows and columns together).

In Bat Pavilion, custom CSS creates the entire red brand identity. CSS variables like --red: #dc2626 ensure colour consistency throughout. The card hover effect (lift + shadow), the hero gradient overlay, the section title underline, and the smooth transitions on buttons are all achieved with CSS transition and transform properties.

CSS Variables Flexbox Hover & Transitions transform: translateY backdrop-filter box-shadow ::after pseudo-element linear-gradient

Bootstrap Framework — Responsive Design Made Easy

Bootstrap is the world's most popular open-source CSS framework. It provides a large collection of ready-made components — navbars, cards, buttons, forms, badges, alerts — along with a powerful 12-column responsive grid system that allows layouts to automatically adapt to different screen sizes.

Bootstrap uses utility classes like d-flex, mt-4, text-muted, and fw-bold to apply styles directly in HTML without writing custom CSS. Its breakpoint system (sm, md, lg, xl) makes responsive design straightforward — for example col-md-4 means "take up 4 columns on medium screens and larger, full width on small screens."

Bat Pavilion uses Bootstrap 5.3 extensively — the fixed <navbar> with mobile hamburger toggle, the 3-column product grid, the custom order form with validation, the alert success message, and the border-start border-danger review cards are all Bootstrap components.

Grid System (col-md-4) Navbar + Toggler Cards Form Controls Buttons Alert Component Utility Classes Responsive Breakpoints
Bootstrap vs Custom CSS — Key Differences

⚡ Bootstrap

  • Pre-written classes — fast to build
  • Consistent across browsers
  • Large community & documentation
  • Built-in responsive grid
  • Less unique visual output

🎨 Custom CSS

  • Full creative control
  • Unique brand aesthetics
  • Smaller file size
  • Requires more time to write
  • No dependency on external library
Bat Pavilion uses both — Bootstrap for the grid, navbar, forms and alerts; custom CSS for the red brand theme, card hover effects and hero gradient.

JavaScript — Making the Web Interactive

JavaScript (JS) is the programming language of the web. While HTML provides structure and CSS provides style, JavaScript adds behaviour — it makes pages dynamic and interactive by responding to user actions in real time, without reloading the page.

JavaScript can manipulate the DOM (Document Object Model) — the live representation of the HTML in the browser — to add, remove, or change elements dynamically. It uses event listeners to respond to user actions like clicks, key presses, and form submissions.

Bat Pavilion uses JavaScript for three key features: (1) a live search that filters bat cards as you type using addEventListener('keyup'), (2) form validation and submission handling that shows a success message without a page reload using e.preventDefault(), and (3) smooth scroll on the "Order Now" buttons using scrollIntoView().
